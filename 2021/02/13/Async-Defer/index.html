Async / Defer 비교 (Defer를 사용해야 하는 이유)

1. head > script

head 안에 그냥 script를 사용하는 경우

  • 처리과정

  • Issue
    js 실행 후 나머지 HTML parsing 하게 된다.
    js 다운로드가 오래걸리는 경우, page loading이 느려짐.

2. body > script

  • body 마지막에 script 를 넣어주는 경우

  • 처리과정

  • Issue
    HTML parsing 완료 후, js 실행
    HTML 로딩은 빠르지만, js에서 동적으로 처리하는 부분을 제대로 볼 수 없음.
    js fetching/executing 시간을 추가로 더 기달려야함.

3. head + async

  • head 안에 script 에서 async를 사용하는 경우

  • 처리과정

  • Issue
    js fetching 시간은 절약.
    뒷부분 HTML 요소가 정의되기 전에 js가 실행 됨.
    js 실행이 오래결리는 경우, page loading 이 느려짐.

4. head + defer (** 이상적인 방법 **)

  • head 안에 script 에서 async를 사용하는 경우
  • 처리과정

5. async / defer 차이 (부가설명)

  • 여러개의 js 파일을 불러오는경우
    async : fetching 되는 js 순서대로 실행 함. (!순서의존적인 경우, 문제발생)
    defer : 선언된 js 순서대로 실행 함.
